# Contributing

## Branch naming
- `feature/<short-desc>` — new work
- `fix/<short-desc>` — bug fixes
- `docs/<short-desc>` — documentation only

## PR checklist
- [ ] PR title explains the change
- [ ] No data files committed
- [ ] No API keys committed
- [ ] Notebooks are personal or coordinated
- [ ] One teammate review before merge

## Notebook rules
- Each person owns their notebook: `notebooks/<name>_*.ipynb`
- Prefer moving reusable code into `src/` after experiments stabilize
